
# 需求提交系统

### 技术栈

前端：vue3+element.plus+js+pinia

后端：java MySQL springboot myb  

#### 三个成员

* 老师
* 管理员
* 干事

## 核心业务流程

```Mermaid
graph TD
    A[老师提交需求] --> B[任务状态：待受理]
    B --> C[管理员分配任务给干事]
    C --> D[任务状态：进行中]
    D --> E[干事受理任务]
    E --> F[干事提交任务完成证明]
    F --> G{管理员审核}
    G -- 通过 --> H[任务状态：已完成]
    G -- 不通过/打回 --> I[任务状态：已打回]
    I --> F
    H --> J[老师进行满意度反馈]
```

**流程说明**：

1. 老师创建任务，状态为 待受理。
2. 管理员将任务分配给具体干事，状态变为 进行中。
3. 干事“受理”任务（这是一个确认动作）。
4. 干事完成工作后，提交成果。
5. 管理员审核干事提交的成果：通过则任务已完成；不通过则打回，干事需重新提交。
6. 任务已完成后，老师可进行反馈。

## 老师功能

1.提交需求（任务名称，详细任务，要求时间，联系方式，其他）

* 任务名称 文本 必填
* 详细任务 长文本（要指定是哪一个部门来完善 然后给一个下拉框可以选择提交给哪一个部门的管理员）必填
* 要求时间 必填（需要一个日期选择器和时间选择器 期望完成时间）
* 联系方式（姓名） 办公地址 必填
* 上传附件 选填
* 提交完之后 表单跳转到任务列表页 并且提示已完成提交

2.任务列表（历史任务，正在进行的任务，删除任务，任务详细信息，反馈任务，任务状态，总理事联系方式）

* 任务列表：“历史任务”，“正在进行的任务” 需要一个明确的需求筛选规则（根据“任务状态”来筛选）
* 任务状态枚举：明确定义任务有哪些状态 比如待受理，进行中，已完成
* 删除任务功能：进行中的任务不可删除 选择逻辑删除（防止误删）
* 任务详细信息：需要明确标出是哪一个部门处理 理事是谁（负责人）负责人联系方式 任务状态 
* 反馈功能（任务完成满意度）：在已完成的任务表单中增加一个反馈的窗口 使得管理员可以接收到老师的反馈

## 管理员

1.同一个样式 渲染不同部门管理员的主页面（从后端账号数据中获取管理员所在部门信息）

2..任务列表（接收老师的需求任务）

* 任务信息：“历史任务”，“正在进行的任务” 需要一个明确的需求筛选规则（根据“任务状态”来筛选）（可以复用老师那边的任务列表）

* 详细任务 ：可以看到老师所提出的需求任务 要求在此表单来分配任务给干事

* 反馈列表：可以看到老师对于已经完成的任务的反馈

* 修改任务状态功能：可以修改该任务的状态 比如待受理，进行中，已完成，已延期，已关闭

* 分配干事任务列表：

  1.需要看到管理员分配给干事的任务

  2.任务状态：未受理 处理中 已提交

  3.打回任务功能：若对干事的任务完成不满意可以打回

3.分配任务给干事（可以选择干事）

* 要求在任务列表中的详细任务表单中使用一个按钮（分配任务）来打开分配任务表单

* 在分配任务表单中需要有

  管理员联系方式 必填

  一个任务分配人数选择器（来决定分配的人数）优先级最高 必填

  一个下拉框（通过前面的人数选择器来选择干事并且干事人数不能超过人数选择器中已经决定的人数）必填

  一个日期时间选择器（用来设置任务完成的截止时间）必填

  一个文本域（来给干事传输被分配任务所要做的详细信息）必填

  一个上传附件功能 选填

* 在表单的中间最下方需要一个“分配”按钮 点击完要提醒“确定分配任务给该干事吗？” 

## 干事

1.查看获取到的任务列表（只需要看见管理员分配的任务即可，若没有任务需要，则显示“任务都已完成！”）

* 需要有状态规则来划分：已完成的任务，未处理的任务，被打回的任务,受理中的任务

* 点击任务弹出该任务详细信息表单

  1.任务详情（任务名称 截止时间 管理员联系方式）

  2.管理员传输给干事的任务要求 也就是上文管理员分配任务表单中的文本域中的信息

  3.未受理的任务详情的居中最下方需要有一个受理的按钮 点击之后将任务状态改成受理中的任务

  4.受理中的任务详情的居中最下方需要有一个提交任务的按钮 点击之后弹出一个提交任务的表单

  表单内容包括：

  上传需求完成证明附件功能

  提交按钮（点击玩要提醒“确定提交吗？” ）

  提交完 任务状态需要改成已完成

  5.还需要一个该任务完成的状态流效果：已受理-->进行中-->已完成（-->已被打回)
